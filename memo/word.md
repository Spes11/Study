# Web サイトにおけるレンダリング

ウェブブラウザが HTML、CSS、JavaScript などのコードを解釈して、ユーザーが見ることができるウェブページを生成するプロセス

# CSR（Client-Side Renderiing）

クライアント（Web ブラウザ）側でページの表示やデータの取得・処理を行う。

- #### メリット
  ページ遷移が高速で、ユーザーエクスペリエンス（ユーザーが感じる使いやすさ）が向上する。  
  クライアント側のレンダリングにより、サーバーの負荷を軽減できる。
- #### デメリット

  初回のページロードには時間がかかる場合がある。  
  クライアント側のブラウザのバージョンや設定次第でページが正しく表示されない場合がある。

  # SSR（Server Side Rendering）

  サーバー側でレンダリングを行い、その結果をクライアントに送信して表示する手法。  
  バックエンドで ページを事前に組み立て、クライアントにはレンダリング済みの HTML が送られる。

- #### メリット
  初回のページロードが早い  
  クライアントのブラウザの設定やバージョンに左右されにくい。
- #### デメリット

  ページ遷移がクライアント側よりも遅い場合がある。  
  サーバーの負担が増加する可能性がある。
